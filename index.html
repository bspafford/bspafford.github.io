Idle Fisher

Idle Fisher is my biggest and most ambitious project to date. It's an idle game built around fishing, upgrading, and idling. It's a pretty simple concept but full of challenges. I'm a solo developer, responsible for all aspects of the game. While I initially had help with the artwork, I did not create all the art myself in addition to handling the programming.

This is one of my videos from my YouTube channel. The video above is where I go over all the different ways on how I made my new and imporved water. I discuss all the different ways of how I went about this, and all the challenges I faced along the way. There are many other videos like this on my channel. They all revolve around Idle Fisher, and me walking you through my through process and challenges of what its like when adding features to a video game

Why

When I started this project, I wanted to push myself beyond the boundaries of my previous work. I had made several smaller, unfinished games using traditional game engines, but this time I wanted to both challenge myself and finish a project. To do that, I decided to build the game without a game engine. I chose to write everything in C++. I originally started with the SDL2 library, but am now transitioning the project to use GLFW, Glad, and GLM to support more advanced graphical features like lighting, shadows, and more realistic water.

Learned

Through this project, I've gained a much deeper understanding of programming, project organization, and even how computers work at a lower level. Since I'm not using a game engine, I've had to build many systems from scratch, systems that are usually handled for you in popular engines. This includes writing custom classes for animations, UI components like buttons and text, I/O like save/load features, file parsers, and more. Rebuilding these core features has given me a strong appreciation for how game engines work under the hood and has significantly improved my coding skills, especially in areas like orginization, debugging, and optimization strategies.

Challenges

One of the biggest challenges in developing Idle Fisher has been balancing depth with simplicity. I've worked to create gameplay that feels rewarding and engaging over time without overwhelming the player or breaking the "idle" experience.
